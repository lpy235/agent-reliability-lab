# Agent Reliability Lab v0.2.0

Agent Reliability Lab v0.2.0 turns the first public demo into a CI-friendly reliability loop. Eval reports now have machine-readable JSON output, Docs QA runs are gated against a checked-in baseline, and the dashboard can inspect report health alongside saved traces.

## Highlights

- GitHub Actions uploads reliability artifacts from every CI run.
- JSON eval reports for Docs QA and Issue Triage suites.
- `arl-baseline` CLI for comparing JSON eval reports and failing on regressions.
- Docs QA baseline gate in CI using `baselines/docs_qa_eval_report.json`.
- Markdown and JSON baseline comparison reports.
- Dashboard Reports view for eval summaries, case tables, and baseline changes.
- Updated dashboard screenshot showing the report inspection workflow.

## Quick Start

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
python -m pytest -v
arl-harness
```

## Compare A Baseline

```bash
arl-baseline baselines/docs_qa_eval_report.json reports/eval-report.json \
  --report-path reports/baseline-comparison.md \
  --json-report-path reports/baseline-comparison.json
```

The command exits with a nonzero status when a shared case moves from passing to failing.

## Try The Reports Dashboard

```bash
arl-api --reload
```

Then open:

```text
http://127.0.0.1:8000/dashboard
```

Use the Reports tab to inspect Docs QA, Issue Triage, and baseline comparison summaries. Set `AGENT_RELIABILITY_REPORTS_DIR` if the reports live outside the default `reports/` directory.

## CI Artifact

The GitHub Actions workflow uploads `agent-reliability-lab-reports` with:

- Markdown eval and baseline reports
- JSON eval and baseline reports
- `runs.db`

This makes each CI run inspectable after it finishes.

## Verification

## Known Limitations

- CI currently gates the Docs QA baseline only; Issue Triage evals are generated and uploaded but not baseline-gated.
- The included agents are deterministic demos, not production issue-triage or RAG systems.
- The dashboard is intended for local inspection and does not include authentication.
- Issue triage records dry-run tool calls only and does not mutate real GitHub issues.
