# Agent Reliability Lab v0.1.0

Agent Reliability Lab v0.1.0 is the first public release of a lightweight reliability toolkit for LLM agents. It makes local agent behavior traceable, replayable, diffable, and regression-testable with a small Python stack.

## Highlights

- Trace SDK and SQLite run storage for agent observability.
- Docs QA demo agent with local-document retrieval and groundedness checks.
- Issue Triage demo agent with deterministic labels, priorities, and dry-run tool calls.
- JSONL eval runner with Markdown reports for Docs QA and Issue Triage cases.
- Replay and diff tools for comparing saved agent behavior.
- PII redaction and tool-policy checks for safety-oriented evals.
- FastAPI API and framework-free dashboard for inspecting runs and traces.
- CLI packaging with `arl-harness`, `arl-eval`, `arl-replay`, `arl-diff`, and `arl-api`.
- Bundled demo docs and eval cases for a reproducible first run.

## Quick Start

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"
python -m pytest -v
arl-harness
```

## Try The Dashboard

```bash
arl-api --reload
```

Then open:

```text
http://127.0.0.1:8000/dashboard
```

## CLI Commands

- `arl-harness`: run the local demo and both bundled eval suites.
- `arl-eval`: run a JSONL eval file and write a Markdown report.
- `arl-replay`: replay a saved Docs QA run with fixed or live retrieval context.
- `arl-diff`: compare two saved runs and optionally write a Markdown diff report.
- `arl-api`: start the FastAPI API and browser dashboard.

## Known Limitations

- The included agents are deterministic demos, not production issue-triage or RAG systems.
- The dashboard is intended for local inspection and does not include authentication.
- Safety checks are lightweight and focus on observable redaction and dry-run tool policy violations.
- Issue triage does not mutate real GitHub issues.
